Wetlandcare Australia

“An alarming 50% of the world’s wetlands have been destroyed in the last 100 years, threatening human welfare at a time of increasing water scarcity.” - United Nations Environment Program

Project Overview

The scope of this project included the re-design of the current website and the creation of a mobile optimised view.

Why

The existing website is outdated and needs a refresh. I does not portray the organisation in a positive light.

What

We needed to understand the requirements of a non-profit. we conducted research, prototyped and tested our ideas.

Research

The research phase was kicked off by a round of guerrilla testing of the current website.

The key findings from the testing then transformed into sticky notes in Miro and then re-arranged into an affinity map.

We then conducted an ideation exercise using the ‘I like, I Wish, What if” method. These ideas were than assessed using a feature prioritisation matrix.

Information Arcitecture

With the key features defined, it was time to lock down the information architecture. This began with a card sorting exercise.

This mapped out all of the pages on the existing website, including links to other pages. we found a lot of duplication of information and broken links.
The categories were grouped and duplicates removed.
The new sitemap was then created.

A user flow was also created to show the path of a typical user. The screens required for this path would become the prototype.

Low Fidelity Design

Using the user flow and sitemap 4 key pages were created as wireframes, the homepage, case-studies, the donate page and a page to show ‘where the money goes’.

A number of elements were created as components such as the header and footer, case study, donate modal and money animation pie chart.

These pages were then linked together in Figma to create the first prototype.

UI Design

In our initial research we asked users ‘ What colours do you suggest are appealing to see in a Wetlands Non-profit organization website?’

The overwhelming answers were blue, green and natural colours. We used this input to create a colour pallet and a style guide to suit including fonts, icons and buttons.

This was then applied to the wireframes to create high fidelity prototypes.

At this stage, we also created the first prototype of the mobile-view version of the site, stacking elements vertically and optimising text sizes and buttons for mobile.

Testing And Iteration

We tested the web prototypes at the wireframe stage and then again at high fidelity alongside the mobile view prototypes. This resulted in a number of findings relating to navigation, semantics of instructions and sizing/visibility of elements.

These updates were made, resulting in the final design.
